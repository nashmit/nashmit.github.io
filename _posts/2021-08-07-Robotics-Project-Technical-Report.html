AbstractThis report presents three different ways to imple-
ment Inverse Kinematics that is applicable in a context of a
serial robot manipulator and the limitations that each of them
has. The solutions chosen are based on Numerical Jacobian,
Denavit-Hartenberg Algorithmic Jacobian, FABRIK and Cyclic
Coordinate Descent.

I. INTRODUCTION
The main scope of this Robotics Practical project is the
analysis and implementation of different approaches to solve
the Inverse Kinematics problem for an open loop chain
robotic arm.
Inverse Kinematics (IK) is dened as the problem
of determining a set of appropriate joint congurations for
which the end-effectors move to desired positions as smoothly,
rapidly, and as accurately as possible. This problem nds
its application in the area of robotics, computer animation,
ergonomics and the computer games industry.
During the last decades, several methods and techniques,
sophisticated or heuristic, have been presented to produce fast
and realistic solutions to the IK problem. However, most of
the currently available methods suffer from high computa-
tional cost, production of unrealistic poses and/or numerical
instability. The aim of this report is to present in full the
most popular approaches and to underline the strength and
weaknesses of each of them. Moreover, Denavit-Hartenberg
Algorithmic Jacobian will be introduced as an intermediate
step for computing Inverse Kinematics [3].
II. REPORT STRUCTURE
The report structure is as follows: In the rst part, we
cover the project setup and the tools used for this project.
We then continue with describing the robot hierarchy. For
this project, we have used the internal structure provided

we develop this concept by describing the FABRIK algorithm
and the CCD algorithm. Immediately after, we introduce the
Jacobian-based approaches I.K. and the Denavit-Hartenberg
Algorithmic Jacobian. Finally, we nish this report with a short
comparison between them.
III. PROJECT SETUP
A. RBDL
Rigid Body Dynamics Library (RBDL) is a self-contained
free open-source software package that implements state of
the art dynamics algorithms including external contacts and
collision impacts. It is based on Featherstones spatial algebra
notation and is implemented in C++ using highly efcient data
structures that exploit sparsities in the spatial operators. The
library contains various helper methods to compute quantities,
such as point velocities, accelerations, Jacobians, angular and
linear momentum and others. We use the library as a proxy
to interact with predened robots modeled with LUA. [7]
B. MeshUp
MeshUp is an open-source software that was developed
at the University of Heidelberg. It is used to visualize and
animate 3D rigid-body systems, render models in real-time
and allows to render motions directly to videos or image
sequences. The models are stored in .lua les which dene
a hierarchy of joints. Each joint is, inter alia, given a mesh, a
parent frame, rotational and/or prismatic degrees of freedom
and a local reference frame [6].
